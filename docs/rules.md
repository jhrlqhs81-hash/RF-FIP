# Docs Rules

- Do not store raw command logs in `docs/`.
- Do not store secrets, keys, or private endpoint values.
- Put durable decisions in `ADR/`.
- Put step execution status in `phases/` or `.codex/context/`, not product specs.
- Keep each doc focused on one contract area.
- Update `REGRESSION_CHECKLIST.md` when a new recurring regression class is found.
